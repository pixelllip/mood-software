## 2. 项目标志 (Project Identity)
* **项目名称**：AI 学习助手 (Learning Assistant)
* **核心目标**：结合学生成绩数据，通过 AI Agent 自动化生成个性化的学习日程，并提供智能答疑服务。
* **技术栈**：Python, PySide6 (UI), OpenAI-compatible SDK (AI Backend), File-based persistence (JSON/TXT).

---

## 3. 思维导图 (Mind Map)

* **AI 聊天**：流式文本输出、上下文记忆、API 状态监控。
* **日程安排**：根据成绩生成、今日/历史日程查看、自动化提醒。
* **成绩管理**：CRUD 操作（增删改查）、模糊匹配搜索。
* **系统设置**：深浅主题切换、持久化配置管理。

---

## 4. 类图 (Class Diagram)

### 核心类说明：
* **`MyWindow` (Frontend)**: 主窗口类，负责 UI 初始化、信号槽路由及各功能页面的切换。
* **`AI_Agent` (Backend)**: 处理 AI 逻辑的核心，包含 API 调用及工具链集成。
* **`ScheduleGenThread` (Worker)**: 异步执行类，防止生成长篇日程时阻塞 UI 线程。
* **`StudentScoreService` (Data)**: 封装成绩数据的存储与逻辑校验。

---

## 5. 功能描述

### 5.1 前端功能 (Frontend)
* **多页容器 (QStackedWidget)**：
    * **聊天页**：支持 Enter 发送/Shift+Enter 换行的文本框，实时显示 AI 生成状态。
    * **历史记录页**：通过 QCalendarWidget 选取日期，从本地 JSON 加载并展示该日的对话摘要。
    * **日程页**：
        * “计划模式”：输入学生信息和作息，调用 AI 生成。
        * “今日模式”：查看已保存的 `schedule_YYYY-MM-DD.txt`。
    * **成绩页**：分标签页处理查询、添加、删除操作，支持 QFormLayout 表单输入。
* **交互细节**：
    * **日历路由**：根据当前页面自动启用/禁用日期选择功能。
    * **响应式布局**：基于 QVBoxLayout 和 QHBoxLayout 实现窗口缩放适配。

### 5.2 后端逻辑 (Backend)
* **AI 驱动引擎**：
    * 基于 `generate_tomorrow_study_schedule` 方法，根据学生成绩（弱势学科）自动分配复习权重。
    * **智能过滤**：生成今日剩余日程时，自动剔除已复习的科目。
* **数据持久化**：
    * **日程存储**：采用 `Schedules/` 文件夹管理 txt 文件。
    * **配置存储**：`settings.json` 记录主题偏好和提醒设置。
    * **对话存档**：以 JSON 格式按日期记录用户与 AI 的交互。
* **线程安全**：使用 QThread 处理耗时任务（如 LLM 请求），通过 Signal/Slot 机制安全地更新 UI 文本。

## 6. 验收标准 (Acceptance Criteria)

### 6.1 性能标准
* **启动速度**：程序启动至主界面显示应在 2 秒内。
* **流式响应**：AI 聊天输出的文字首包延迟应低于 1.5 秒（视网络情况而定）。
* **UI 流畅度**：生成日程期间，用户应能正常点击其他导航按钮（非阻塞）。

### 6.2 逻辑标准
* **日期限制**：不允许为过去的日期生成新的学习计划。
* **提醒机制**：若 `settings.json` 中开启了提醒，启动时必须检测今日是否有未读日程。
* **健壮性**：若本地 JSON 损坏，程序应能加载默认配置而非崩溃。

### 6.3 视觉标准
* **主题一致性**：切换为“朦胧灰”或“纯净白”时，所有控件（包括日历和下拉框）的边框与背景色必须同步更新。
