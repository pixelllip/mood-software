1. 项目标志 (Project Logo & ID)项目名称：Aegis Academic (星火学伴)版本号：v1.0.0-Beta核心定位：基于异步 AI 技术的本地化智能学业管理工作站。2. 迭代记录 (Iteration Record)版本号修订日期修订内容负责人v0.12026-03完成 AI Agent 核心调用与 PySide6 基础 UI 构建开发团队v0.52026-04集成 StudentScoreService，实现成绩 CRUD 功能开发团队v1.02026-05(规划中) 增加基于成绩数据的规划生成算法界面开发团队3. 思维导图 (Mental Map)该项目逻辑架构分为以下四个维度：交互层：流式 AI 聊天、历史记录追溯（日历关联）。数据层：本地成绩档案管理（班级/学号/姓名/课程/分数）。规划层 (拟开发)：成绩分析引擎、学习路径规划。支撑层：异步多线程架构、事件过滤器机制。4. 类图 (Class Diagram)核心类关系描述：MyWindow (QWidget)：主控界面，负责 UI 初始化与信号分发。AI_Agent (QThread)：负责异步 API 调用及流式信号输出。StudentScoreService：负责成绩数据的业务逻辑处理与持久化。MySlot：事件响应中心，处理 UI 与逻辑层之间的交互。5. 功能描述 (Functional Description)异步流式聊天：通过 QThread 解决 AI 思考时的界面假死问题，支持实时文本追加。结构化成绩管理：支持通过正则解析快捷录入成绩，并提供模糊搜索功能。时间维度回溯：点击日历控件可动态加载历史对话日志。智能规划生成 (Next)：读取本地成绩，由 AI 生成个性化的提升建议。6. 验收标准 (Acceptance Criteria)性能体验：AI 响应过程中 UI 保持 60FPS 顺滑，无卡顿感。交互规范：支持 Enter 发送、Shift+Enter 换行，符合主流 IM 软件操作习惯。鲁棒性：网络断开或 API Key 错误时须有明确的异常弹窗提醒。数据一致性：成绩的增删改查需实时同步至后台数据结构，查询结果无延迟。
